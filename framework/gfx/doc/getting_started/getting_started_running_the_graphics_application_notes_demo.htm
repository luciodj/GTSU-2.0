Running the Application Notes Demo:

This demo will initially show the starting screen that will allow users to choose to run internal resource, external resources or program the external memory. If one of the first two choices is made, the demo will run showing the main screen of the demo. The main screen shows folder icons for each application note. Press on a folder icon to launch a specific application note demo. Each application note will have a back button to go back to the main icon screen.

If the programming of external memory is selected, the external hex file to program is found in the source directory of the demo: <mla_root_folder>/apps/gfx/app_notes/firmware/src.  The file is named: “external_resource_main.hex”.

 

Running the Graphics AN1136 Demo:

This demo uses the selected hardware platform to show how Widgets are used in an application. Two buttons on the screen controls a slider and draws multiple bars to emulate a volume or a level. The buttons and slider can be controlled by the user through the touch screen. A user press on one of the buttons will move the slider and change the volume level. A touch on the left button moves the slider to the left and reduces the level. A touch on the right button moves the slider to the right and reduces the level.

 

Refer to the application note “How to Use Widgets in Microchip Graphics Library (AN1136)” downloadable in www.microchip.com/graphics under the Training and Support section for details.

 

Running the Graphics AN1182 Demo:

This demo uses the selected hardware platform to show how Fonts are used in an application. Two buttons on the screen controls an Edit Box contents. The Edit Box controls the phrase “Hello World” to be displayed in different languages (fonts).

 

Refer to the application note “Fonts in Microchip Graphics Library (AN1182)” downloadable in www.microchip.com/graphics under the Training and Support section for details.

 

Running the Graphics AN1227 Demo:

This demo uses the selected hardware platform to show how Microchip Graphics Library simplifies the design of a keyboard based GUI.

 

Refer to the application note “Using a Keyboard with the Microchip Graphics Library (AN1227)” downloadable in www.microchip.com/graphics under the Training and Support section for details.

 

Running the Graphics AN1246 Demo:

This demo and the accompanying application note serves as a useful guide in creating customized Widgets. The essential components of a Widget are enumerated and described in the application note as well as the process of integrating the new Widget into the Graphics Library.

 

Refer to the application note “How to Create Widgets in Microchip Graphics Library (AN1246)” downloadable in www.microchip.com/graphics under the Training and Support section for details.

Note:

1.     When using PIC24FJ256DA210 Development Board set the jumpers to:

-       JP7 – close (use USB device port if programming external memory)

-       JP8 – open (color depth is 16BPP, byte access is not performed)

-       JP9 – set to 1-2 (use Y- signal)

-       JP10 – set to 1-2 (use X+ signal)

-       JP11, JP12 – don’t care

-       JP13 – set to RG8-S1 (use S1 switch)

-       JP14 – set to RE9-S2 (use S2 switch)

-       JP15 – set to RB5-POT(RB5) (use potentiometer)

-       JP23 – set to 2-3 (use SPI Flash)

2.     When using Graphics LCD Controller PICtail™ Plus SSD1926 Board set the jumpers to:

-       JP1 – open

-       JP2 – depending on the PMP interface setting

- set to 1-2 (for 8-bit PMP)

- set to 2-3 (for 16-bit PMP)

-       JP3 – set to 1-2 (use RD1)

3.     To run this demo on the 5.7″ VGA Truly TFT Display Board (AC164127-8) with the PIC24FJ256DA210 Development Board at 16 bpp color depth, the refresh rate of the display must be set to 32 Hz instead of 60 Hz.

External Memory Programming

 

Programming the hex files to the external memory can be performed using the “memory_programmer.jar” utility located at

<mla_root_folder>/framework/gfx/utilities/memory_programmer. Please refer to the help file of the utility for details of usage.

 

For additional details on setting up the external memory programming mode, see ‘getting_started_programming_external_memory’.

 

The table below shows the programming mode used for each configuration of the project.

·         UART means, the UART port of the PIC device is used to send external hex data. Default baud rate is 115200.

·         USB means, the PIC device will act as a USB Device and external data will be sent through the USB Device port.

To program the external memory, select the option at the start of the demo that says: “Program External Resources”.

Once that option is selected, the firmware will proceed to the programming mode and wait for the data to be sent from the host computer.

The hex file to use in this demo is located in:

<mla_root_folder>/apps/gfx/app_notes/firmware/src.

The file is named: “external_resource_main.hex”.

 

After the external memory is programmed, the demo will restart automatically.

Graphics Utilities Location

 

The Graphics Resources used in this demo was created using the Graphics Resource Converter (GRC).

The Graphics Resources located in the external memory was programmed using the External Memory Programmer Utility.

Both of these utilities can be found in:

Folder Path for utility, Graphics Resource Converter: <mla_root_folder>/framework/gfx/utilities/grc

Folder Path for utility, External Memory Programmer: <mla_root_folder>/framework/gfx/utilities/memory_programmer
